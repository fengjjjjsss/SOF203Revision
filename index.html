2. Layered Models and Responsibilities 分层模型与各层职责

Why layering? / 为什么要分层

English (memorize) / 英文背诵：Layering gives an explicit structure for identifying the pieces of a complex network system and their relationships. Modularization simplifies discussion, maintenance, and updating because a layer can change its implementation while keeping the same service interface to other layers.

中文理解：分层为复杂网络系统提供明确结构，使各组成部分及其关系可被识别。模块化便于讨论、维护和升级；只要层间服务接口不变，某一层的内部实现可以改变而不影响其他层。

Names of the 5-layer and 7-layer models / 五层与七层名称

English (memorize) / 英文背诵：The Internet model is a descriptive five-layer model used in practice. The ISO/OSI model is a prescriptive seven-layer model. The OSI presentation and session functions are normally implemented inside the application layer of the Internet stack when needed.

中文理解：Internet 模型是实践中使用的描述性五层模型；ISO/OSI 是规范性的七层模型。OSI 的表示层和会话层功能，在 Internet 协议栈中如有需要通常由应用层实现。

Responsibilities of each Internet layer / Internet 五层职责

The two extra OSI layers / OSI 多出的两层

Presentation layer / 表示层: enables applications to interpret data; may perform translation, encryption/decryption, compression/decompression, and machine-specific representation.

Session layer / 会话层: manages dialog sessions and provides synchronization, checkpoints, and recovery of an interrupted data exchange.

3. Encapsulation and Decapsulation 封装与解封装

English (memorize) / 英文背诵：Encapsulation is the downward process in which each protocol layer adds its own control information, usually a header, to the data received from the layer above. Decapsulation is the upward process in which each receiving layer interprets and removes the corresponding control information before passing the payload upward.

中文理解：封装是数据向下经过协议栈时，每层向上层交付的数据添加本层控制信息（通常是首部）的过程。解封装是接收端数据向上经过协议栈时，每层解释并移除相应控制信息，再把有效载荷交给上层的过程。

What happens at intermediate devices? / 中间设备怎样处理

A switch normally examines the link-layer header and forwards a frame using MAC addresses. It does not remove the end-to-end transport information.

A router removes the incoming link-layer frame, examines the IP datagram, selects an outgoing interface, and encapsulates the same IP datagram in a new link-layer frame for the next hop.

Therefore, source and destination IP addresses normally remain end-to-end, while source and destination MAC addresses change at every routed hop. TTL is updated by routers; IP addresses at router">NAT is a special case that can rewrite IP addresses.

课件来源 / Slide source: SOF203-Chap1-Introduction.pdf, PDF pp. 53-60; SOF203-Chap3-Link Layer - Part 1.pdf, PDF pp. 21-26.

4. Multiplexing: FDM and TDM 复用：频分与时分

English (memorize) / 英文背诵：Multiplexing allows multiple signals to travel simultaneously over one physical medium by dividing it into logical subchannels. A multiplexer combines input channels at the sender, and a demultiplexer separates them at the receiver.

中文理解：复用把一条物理介质划分为多个逻辑子信道，使多个信号共享介质。发送端的复用器把多个输入合并，接收端的解复用器再把它们分离。

Frequency Division Multiplexing (FDM) / 频分复用

English (memorize) / 英文背诵：FDM divides the frequency spectrum of a link into separate frequency bands. Each connection is assigned one band and can transmit at the same time as the others. It is used in broadcast radio and cable television.

中文理解：FDM 把链路的频谱划分为互不重叠的频带。每条连接占用一个频带，因此各连接可同时传输。广播电台和有线电视使用这种方式。

Time Division Multiplexing (TDM) / 时分复用

English (memorize) / 英文背诵：TDM divides transmission time into slots. Senders take turns, usually in round-robin order, and each sender transmits during its assigned slot.

中文理解：TDM 把传输时间划分为时隙。发送者按顺序轮流使用介质，通常采用轮询方式，每个发送者在分配给自己的时隙中传输。

6. Circuit Switching vs Packet Switching 电路交换与分组交换

Circuit switching / 电路交换

English (memorize) / 英文背诵：Circuit switching establishes a dedicated end-to-end path before data transfer. Communication has three stages: circuit setup, circuit use, and circuit termination. Once established, its performance is similar to an isolated physical path because resources are reserved for the user.

中文理解：电路交换在传输数据前建立专用端到端路径，过程分为建立电路、使用电路和终止电路三个阶段。由于资源为该用户预留，建立后性能类似独立物理通路。

Packet switching / 分组交换

English (memorize) / 英文背诵：Packet switching divides data into packets and shares links among users. No dedicated circuit setup is required; packets are stored and forwarded by routers. Sharing improves resource utilization, but packets may experience variable queueing delay and may be dropped when buffers are full.

中文理解：分组交换把数据划分为分组，让多个用户共享链路。它不需要建立专用电路，路由器采用存储转发。共享提高资源利用率，但分组会经历可变的排队时延，缓存满时还可能丢失。

Circuit-switched networks provide stable, predictable delay and no packet loss because a dedicated path is reserved end-to-end. This makes them suitable for real-time applications like voice calls.

电路交换提供稳定的、可预测的延迟且无丢包，因为端到端建立了专用路径。适合语音通话等实时应用。

7. Four Sources of Packet Delay 分组的四类时延

English (memorize) / 英文背诵：A packet can be delayed by nodal processing, queueing, transmission, and propagation. These delays have different causes and must not be confused.

中文理解：分组可能经历处理、排队、传输和传播四类时延。四者成因不同，不能混淆。

Transmission delay and propagation delay are very different / 传输与传播时延的本质区别

English (memorize) / 英文背诵：Transmission delay is the time to put the bits onto the link, whereas propagation delay is the time for a bit already on the link to travel to the other end. A long packet increases transmission delay; a long cable increases propagation delay.

中文理解：传输时延是“把比特送上链路”的时间；传播时延是“已经在链路上的比特走到另一端”的时间。分组越长，传输时延越大；链路越长，传播时延越大。

Unit conversion checklist / 单位换算

Packet length: bytes x 8 = bits. / 分组长度：字节乘 8 变成比特。

Distance: kilometres x 1,000 = metres. / 距离：千米乘 1,000 变成米。

Rate: 1 Mbps = 10^6 bps; 1 Gbps = 10^9 bps. / 速率使用十进制单位。

Time: 1 s = 1,000 ms = 10^6 microseconds. / 秒、毫秒和微秒要统一。

Worked exam example / 复习题例题

A 2,000-byte packet travels over a 2,500 km link. Propagation speed is 2.5 x 10^8 m/s and transmission rate is 100 Mbps.

Convert: L = 2,000 x 8 = 16,000 bits; d = 2,500 x 1,000 = 2.5 x 10^6 m; R = 100 x 10^6 bps.

Propagation delay: d_prop = d/s = (2.5 x 10^6)/(2.5 x 10^8) = 0.01 s = 10 ms.

Transmission delay: d_trans = L/R = 16,000/(100 x 10^6) = 0.00016 s = 0.16 ms.

8. ARP and Packet-Trace Analysis ARP 与抓包分析

English (memorize) / 英文背诵：The Address Resolution Protocol (MAC address on same LAN">ARP) maps a known IPv4 address to the MAC address of an interface on the same LAN. Each host or router maintains an MAC address on same LAN">ARP table containing IP-to-MAC mappings and a TTL.

中文理解：地址解析协议 MAC address on same LAN">ARP 把已知 IPv4 地址解析为同一 LAN 上接口的 MAC 地址。每台主机或路由器维护 MAC address on same LAN">ARP 表，其中包含 IP-MAC 映射及其生存时间 TTL。

Same-LAN request and reply / 同一 LAN 内的请求与应答

The sender checks its MAC address on same LAN">ARP cache. If a valid mapping exists, it uses that MAC address directly. / 发送者先查 MAC address on same LAN">ARP 缓存。

If the mapping is missing, the sender broadcasts an MAC address on same LAN">ARP Request in an Ethernet frame whose destination MAC is FF-FF-FF-FF-FF-FF. The request asks: Who has target IP? / 若无映射，广播 MAC address on same LAN">ARP Request。

Every node on the LAN receives the request, but only the interface whose IP matches the target IP replies. / LAN 上所有节点收到请求，只有目标 IP 的拥有者应答。

The target sends an MAC address on same LAN">ARP Reply by unicast to the requester's MAC address, because the requester included its own MAC and IP in the request. / 目标以单播把 MAC address on same LAN">ARP Reply 发回请求者。

The requester caches the mapping until it times out. MAC address on same LAN">ARP is plug-and-play and the table is soft state. / 请求者缓存映射，过期后删除。

Why broadcast request but unicast reply? / 为什么请求广播、应答单播

English (memorize) / 英文背诵：The request must be broadcast because the sender does not yet know which MAC address owns the target IP. The reply can be unicast because the target learned the requester's MAC address from the request, and only that requester needs the answer.

中文理解：请求必须广播，因为发送者还不知道目标 IP 对应哪一个 MAC；应答可单播，因为目标已从请求中获知请求者的 MAC，而且只有请求者需要答案。

9. Learning Switch: Self-Learning and Forwarding 学习型交换机

English (memorize) / 英文背诵：An Ethernet switch is a transparent link-layer device that stores and forwards frames. Its switch table maps a host MAC address to an outgoing interface and normally includes an ageing time.

中文理解：以太网交换机是透明的链路层存储转发设备。交换表把主机 MAC 地址映射到输出接口，通常还包含老化时间。

Source learning / 源地址学习

English (memorize) / 英文背诵：Whenever a frame arrives, the switch records or refreshes the mapping from the source MAC address to the incoming interface. A switch learns from the source address, not from the destination address.

中文理解：每当帧到达，交换机就记录或刷新“源 MAC - 入接口”的映射。交换机从源地址学习，而不是从目的地址学习。

Destination lookup and action / 查目的地址并处理

Record the source MAC and incoming interface. / 学习源 MAC 与入接口。

Look up the destination MAC in the switch table. / 查询目的 MAC。

If the destination is known on a different interface, forward only on that interface. / 已知且在另一接口：定向转发。

If the destination is known on the same incoming interface, filter/drop the frame because the destination is already on that segment. / 已知且在同一入接口：过滤。

If the destination is unknown, or the frame is broadcast, flood it on every interface except the incoming interface. / 未知或广播：除入接口外泛洪。

10. Active and Passive Scanning in WLAN WLAN 主动与被动扫描

English (memorize) / 英文背诵：Before a wireless station can exchange network-layer data through an access point (AP), it must discover and associate with an AP. The SSID identifies the wireless network and is carried in management information such as beacon frames.

中文理解：无线站点通过 AP 交换网络层数据前，必须先发现并关联某个 AP。SSID 用来标识无线网络，并出现在 Beacon 等管理信息中。

Passive scanning / 被动扫描

APs periodically transmit Beacon frames. / AP 周期性发送 Beacon。

The station listens without first transmitting and selects an AP. / 站点先监听，不主动发探测帧。

The station sends an Association Request to the selected AP. / 向选中的 AP 发送 Association Request。

The AP returns an Association Response. / AP 返回 Association Response。

Active scanning / 主动扫描

The station broadcasts a Probe Request. / 站点广播 Probe Request。

Nearby APs return Probe Responses. / 附近 AP 返回 Probe Response。

The station selects an AP and sends an Association Request. / 站点选择 AP 并发送关联请求。

The AP sends an Association Response. / AP 返回关联应答。

11. Subnetting 子网划分

English (memorize) / 英文背诵：Subnetting divides one large IP network into smaller broadcast domains connected by routers. The key operation is to borrow bits from the host portion and use them as subnet bits.

中文理解：子网划分把一个较大的 IP 网络分成多个由路由器连接的小型广播域。核心操作是从主机位借位，把借来的位作为子网位。

CIDR and address roles / CIDR 与地址角色

In a prefix such as 192.168.123.132/24, /24 means the first 24 bits are network-prefix bits and the subnet mask contains 24 ones.

Network address: all remaining host bits are 0; it identifies the subnet and cannot be assigned to a normal host.

Broadcast address: all remaining host bits are 1; it reaches all hosts in that subnet and cannot be assigned to a normal host.

Usable host range: network address + 1 through broadcast address - 1.

General procedure / 通用步骤

Determine the required number of subnets K and/or hosts per subnet L.

Choose the smallest x such that 2^x >= K. The new prefix is old prefix + x.

Check the remaining host bits h. Under the conventional rule, usable hosts = 2^h - 2.

Convert the new prefix to a dotted-decimal subnet mask.

Find the block size in the interesting octet: block size = 256 - mask value in that octet.

List network addresses in block-size increments; for each subnet find first usable, last usable, and broadcast.

Worked question: 130.56.0.0/16 into 1,024 subnets / 例题

1,024 = 2^10, so borrow 10 bits. New prefix = /16 + 10 = /26.

A /26 mask is 255.255.255.192. Six host bits remain, so each subnet has 2^6 = 64 total addresses and 62 usable host addresses.

Block size = 256 - 192 = 64 in the fourth octet.

12. Dijkstra's Link-State Algorithm Dijkstra 链路状态算法

English (memorize) / 英文背诵：In link-state routing, every router obtains a complete view of the network topology and link costs through link-state flooding. Dijkstra's algorithm then computes the least-cost paths from one source to all other nodes and produces the source router's forwarding table.

中文理解：在链路状态路由中，各路由器通过链路状态泛洪获得完整拓扑和链路代价。随后 Dijkstra 算法从一个源点计算到所有其他节点的最小代价路径，并生成源路由器的转发表。

Notation / 记号

c(x,y): direct link cost from x to y; infinity if x and y are not neighbours.

D(v): current estimate of the least path cost from the source to v.

p(v): predecessor of v on the current least-cost path.

N': set of nodes whose least-cost paths are definitively known.

Algorithm / 算法步骤

Initialize N' with the source. Set D(v) to the direct link cost for each neighbour and infinity for every non-neighbour.

Choose the unvisited node w with the smallest D(w), and add w to N'.

For every unvisited neighbour v of w, update D(v) = min[D(v), D(w) + c(w,v)]. If improved, set p(v) = w.

Repeat until all nodes are in N'. Trace predecessors backward to construct paths and next hops.

Worked revision graph from source t / 从 t 出发的复习题

Edges: t-u=2, t-v=4, t-y=7, u-v=3, u-w=3, v-w=4, v-x=3, v-y=8, w-x=6, x-y=6, x-z=8, y-z=12.

13. Distance-Vector Routing 距离向量路由

English (memorize) / 英文背诵：A distance-vector router knows only its directly connected neighbours, the link cost to each neighbour, and the distance vectors advertised by those neighbours. It repeatedly applies the Bellman-Ford equation until all estimates converge.

中文理解：距离向量路由器只知道直接相邻路由器、到各邻居的链路代价，以及邻居通告的距离向量。它反复应用 Bellman-Ford 方程，直到所有估计值收敛。

Operational logic / 工作逻辑

Initialization: cost to self is 0, cost to each direct neighbour is the direct link cost, and cost to every other node is infinity.

Each node sends its distance vector to its neighbours.

After receiving neighbours' vectors, the node computes every destination through every neighbour using c(a,i)+D_i(b).

If a lower value is found, update the cost and next hop. Send the new vector when it changes.

Continue synchronous rounds or asynchronous triggered updates until no table changes.

Iterative: updates repeat. Asynchronous: a local iteration is triggered by a link-cost change or a neighbour's message. Distributed: nodes use only neighbour information.

Worked three-node example / 三节点例题

Link costs: c(x,y)=2, c(y,z)=1, c(z,x)=4.

Reason / 推导: x learns that x->y->z costs 2+1=3, which is better than the direct cost 4. Symmetrically, z learns z->y->x costs 1+2=3. Node y already has the best direct routes.

14. Identifying IP and MAC Addresses Along a Path 路径中的 IP 与 MAC

Core rule / 核心规则

English (memorize) / 英文背诵：The IP header describes the original end hosts, while the Ethernet header describes the current hop. A router removes the incoming Ethernet frame and creates a new frame for the next link. Therefore, the IP source and destination normally remain unchanged, but the MAC source and destination become the outgoing router interface and the next-hop interface.

中文理解：IP 首部描述原始端系统，Ethernet 首部描述当前一跳。路由器移除入站以太网帧，并为下一条链路创建新帧。因此 IP 源/目的通常不变，而 MAC 源/目的会变成出接口与下一跳接口的 MAC。

Decision method / 判断步骤

Write the original source IP and final destination IP. Keep them unchanged at every listed hop unless the question explicitly introduces IP addresses at router">NAT.

For each link, identify the transmitting interface and the next receiving interface.

Frame source MAC = transmitting interface MAC. Frame destination MAC = next-hop receiving interface MAC.

If the next-hop MAC is unknown, MAC address on same LAN">ARP resolves that next-hop interface; it does not resolve a remote final host.

15. TCP Sequence/Acknowledgment and Reliable Transfer TCP 序号、确认号与可靠传输

TCP characteristics / TCP 特征

English (memorize) / 英文背诵：TCP provides a logical, connection-oriented, full-duplex byte-stream service between one sending process and one receiving process. It uses port numbers to identify processes, establishes a connection by a three-way handshake, transfers data reliably, and tears the connection down.

中文理解：TCP 在一个发送进程与一个接收进程之间提供逻辑的、面向连接的、全双工字节流服务。它用端口号标识进程，通过三次握手建立连接，可靠传输数据，再拆除连接。

MSS (Maximum Segment Size) is the maximum application data carried in one TCP segment. It is smaller than the link MTU because IP and TCP headers also consume space.

A normal TCP header is 20 bytes without options and can be up to 60 bytes. Exam-relevant fields include source/destination ports, sequence number, acknowledgment number, flags, window size and checksum.

Sequence number / 序号

English (memorize) / 英文背诵：TCP numbers bytes, not segments. The sequence number in a data segment is the number of the first data byte carried in that segment. If a segment begins with sequence number S and carries L bytes, the next segment begins with S+L.

中文理解：TCP 给字节编号，而不是给报文段编号。数据报文段的序号是该段第一个数据字节的编号。若某段序号为 S、携带 L 字节，则下一段序号为 S+L。

Acknowledgment number / 确认号

English (memorize) / 英文背诵：The acknowledgment number is the sequence number of the next byte the receiver expects. It is cumulative: ACK=N means every byte through N-1 has been received in order.

中文理解：确认号表示接收方下一步期望收到的字节序号。它是累积确认：ACK=N 表示截至 N-1 的所有字节都已按序收到。

Three-way handshake / 三次握手

Data transfer and port reversal / 数据传输与端口反向

If client data starts at x+1 and carries L bytes, the server returns ACK = x+1+L.

A reply reverses the port numbers: if A->B uses source port 302 and destination port 80, B->A uses source port 80 and destination port 302.

If an out-of-order segment arrives while an earlier byte is missing, the receiver repeats the acknowledgment number of the next missing byte.

Connection teardown / 连接终止

English (memorize) / 英文背诵：Each direction of a TCP connection is closed independently. One side sends FIN, the peer acknowledges it, the peer later sends its own FIN, and the first side acknowledges that FIN. An ACK can be combined with the peer's FIN. The active closer enters TIME_WAIT for about twice the maximum segment lifetime.

中文理解：TCP 的两个传输方向独立关闭。一方发送 FIN，对方确认；对方随后发送自己的 FIN，最初一方再确认。确认可以与对方的 FIN 合并。主动关闭方进入 TIME_WAIT，持续约 2 倍最大报文段寿命。

Reliability foundation: ARQ / 可靠传输基础

English (memorize) / 英文背诵：Automatic Repeat reQuest (ARQ) achieves reliable transfer over an unreliable channel through error detection, receiver feedback, and retransmission. Sequence numbers detect duplicates and ordering; acknowledgments report successful receipt; timers recover from lost packets or lost ACKs.

中文理解：自动重传请求 ARQ 通过错误检测、接收方反馈和重传，在不可靠信道之上实现可靠传输。序号检测重复与顺序，确认反馈成功接收，定时器处理分组或 ACK 丢失。

Stop-and-wait and pipelining / 停等与流水线

Stop-and-wait sends one packet and waits for its ACK. It handles four main cases: no loss, data loss/corruption, ACK loss, and premature timeout/delayed ACK. It is correct but may waste bandwidth on a large bandwidth-delay path.

Pipelining allows several unacknowledged packets to be in flight. It requires a larger sequence-number space and sender/receiver buffering, and it greatly improves utilization.

How TCP provides reliability / TCP 可靠性的具体实现

TCP combines ideas from GBN and SR: pipelined byte-oriented segments, cumulative ACKs, optional selective acknowledgments, checksums and retransmissions.

The sender maintains a retransmission timer for the oldest unacknowledged data. A timeout retransmits the segment that caused the timeout, restarts the timer, and normally increases the timeout interval.

TCP estimates the round-trip time adaptively because RTT differs between connections and changes over time.

Receiver ACK rules: delay the ACK briefly for one in-order segment; immediately ACK when another in-order segment arrives; immediately send a duplicate ACK for an out-of-order segment; immediately ACK when a gap is filled.

Fast retransmit: after three duplicate ACKs for the same missing byte, retransmit the smallest unacknowledged segment without waiting for the timer.

In practice, a receiver normally buffers out-of-order bytes until the missing gap is filled, then delivers the stream in order.

Flow control uses the receiver window rwnd so the sender does not overflow the receiver buffer. Congestion control uses cwnd so the sender does not overload the network. The actual sending window is limited by min(rwnd, cwnd).

Appendix A. Final Revision Questions — Detailed Solutions 复习题精讲（点击展开）

Q1: List the five Internet layers and describe the responsibility of each. / 列出 Internet 五层模型并描述每层职责。

▶
Answer / 答案

EN: Application: message format/meaning and application procedures. Transport: process-to-process transfer, ports, reliability, flow/congestion control. Network: host-to-host IP addressing, routing and forwarding. Link: hop-to-hop frames, MAC and medium access. Physical: bits on the wire, signals and media.

CN: 应用层：消息格式/含义及应用过程。传输层：进程到进程传输、端口、可靠性、流量/拥塞控制。网络层：主机到主机IP寻址、路由与转发。链路层：逐跳帧传输、MAC地址与介质访问。物理层：比特在线路上的传输、信号与介质。

Explanation / 解析

口诀记忆：Application=What（消息格式），Transport=Who（哪个进程），Network=Where（路由到哪），Link=How next hop（下一跳），Physical=Bit on wire（比特怎么传）。区分 Network 的 routing（选路）和 Link 的 forwarding（实际转发）——这是高频混淆点。

Q2: What is the main advantage of a circuit-switched network? / 电路交换网络的主要优势是什么？

▶
Answer / 答案

EN: Reserved resources provide a guaranteed rate and predictable performance without queueing from other users after setup. Trade-offs: setup delay and idle reserved capacity.

CN: 预留资源提供有保证的速率和可预测的性能，建立后不受其他用户排队影响。代价：建立时延和空闲资源浪费。

Explanation / 解析

电路vs分组：电路=租专线（质量稳但浪费），分组=共享（效率高但时延不可预测）。关键词 guaranteed + predictable。

Q3: A 2000-byte packet over 2500km link. Speed=2.5x10^8 m/s, rate=100 Mbps. Find d_prop and d_trans. / 求传播时延和传输时延。

▶
Answer / 答案

EN: L=2000x8=16000 bits; d=2500x1000=2.5x10^6 m; R=10^8 bps. d_prop=d/s=0.01s=10ms. d_trans=L/R=0.00016s=0.16ms.

CN: 统一单位后：d_prop=10ms，d_trans=0.16ms。

Explanation / 解析

三步走：(1)统一单位；(2)d_prop=d/s（距离/速度，和包长无关）；(3)d_trans=L/R（包长/速率，和距离无关）。考试陷阱：把 d_trans 和 d_prop 弄混！

Q4: Why is ARP request broadcast but reply unicast? / 为什么ARP请求广播而应答单播？

▶
Answer / 答案

EN: Request broadcast: sender does not know target MAC. Reply unicast: target learned requester MAC from request, and only requester needs the answer.

CN: 请求广播=不知道目标MAC；应答单播=已知请求者MAC且只需告诉请求者。

Explanation / 解析

ARP逻辑：不知MAC→广播问所有人→只有目标IP回复→单播回。陷阱：远程目标→ARP的是网关MAC不是远程主机MAC！

Q5: PC1(10.1.1.2)->R1->R2->PC3(10.3.3.2). Fill MAC/IP per hop. / 填写每跳MAC和IP。

▶
Answer / 答案

EN: IP src=10.1.1.2, dst=10.3.3.2 (unchanged). MAC: PC1->R1: src=PC1_MAC dst=R1_LAN1_MAC; R1->R2: src=R1_LAN2_MAC dst=R2_LAN2_MAC; R2->PC3: src=R2_LAN3_MAC dst=PC3_MAC.

CN: IP端到端不变，MAC逐跳变。路由器拆帧→查路由→封装新帧。

Explanation / 解析

核心规则：IP=最终目的地(不变)，MAC=当前一跳收发方(变)。新帧源MAC=出接口MAC，目的MAC=下一跳接口MAC。

Q6: Switch receives: B->E, E->B, A->B, B->A. Describe per step. / 描述交换机每步行为。

▶
Answer / 答案

EN: B->E: learn B, flood. E->B: learn E, unicast to B. A->B: learn A, unicast to B. B->A: refresh B, unicast to A.

CN: B->E学B泛洪，E->B学E单播B，A->B学A单播B，B->A刷新B单播A。

Explanation / 解析

口诀：Learn source, lookup dest。已知→单播，未知→泛洪，同端口→过滤。

Q7: 130.56.0.0/16 into 1024 subnets. New prefix, mask, first 4. / 划分1024子网。

▶
Answer / 答案

EN: 1024=2^10, borrow 10. New: /26, mask 255.255.255.192. Block=64. Subnets: .0/.64/.128/.192; bcast: .63/.127/.191/.255; 62 usable hosts each.

CN: /26,掩码255.255.255.192。块大小64。每子网62个可用主机。

Explanation / 解析

步骤：2^x>=子网数→借x位→新前缀→掩码→块大小=256-末字节掩码→网络按块递增→广播=下一网络-1→可用=2^h-2。别忘减2！

Q8: Nodes x,y,z: c(x,y)=2,c(y,z)=1,c(z,x)=4. Run distance vector. / 运行距离向量。

▶
Answer / 答案

EN: Init: x=(0,2,4),y=(2,0,1),z=(4,1,0). R1: x->(0,2,3 via y), z->(3 via y,1,0). R2: converged.

CN: x经y到z=3<4更新；z经y到x=3<4更新。一轮收敛。

Explanation / 解析

Bellman-Ford方程：D_x(y)=min{c(x,v)+D_v(y)}对所有邻居v取最小。x发现2+1=3<4所以更新。考试要展示每轮计算！

Q9: TCP handshake: client_ISN=3974188230, server_ISN=1884869393. Find seq/ack at each step and after 331B data. / 求各步序号/确认号。

▶
Answer / 答案

EN: SYN:Seq=x. SYN/ACK:Seq=y,ACK=x+1. Final ACK:Seq=x+1,ACK=y+1. Data:Seq=x+1. After 331B:ACK=x+332.

CN: 握手：SYN Seq=x；SYN/ACK Seq=y ACK=x+1；ACK Seq=x+1 ACK=y+1。331字节后ACK=x+1+331。

Explanation / 解析

TCP三规则：SYN占1号，FIN占1号，纯ACK不占号。TCP给字节编号不是给段编号！

Q10: Dijkstra from node t on given graph. Show iteration table. / 从t运行Dijkstra。

▶
Answer / 答案

EN: u=2(t-u), v=4(t-v), w=5(t-u-w), x=7(t-v-x), y=7(t-y), z=15(t-v-x-z).

CN: u=2,v=4,w=5,x=7,y=7,z=15。

Explanation / 解析

松弛操作：D(v)=min(D(v),D(w)+c(w,v))。选最近未访问节点→更新邻居→重复。考试必须展示迭代过程！

Appendix B. Last-Minute English Recitation 考前英文速背卡片

16 core concepts condensed into conversational, memorable English — designed for rapid last-minute scan. Read through once and you are ready for definitions questions. / 16个核心概念浓缩为人话版英文卡片，考前快速扫一遍即可应对定义题。

Network Scale
PAN covers one person; LAN covers a building; MAN covers a city; WAN covers a country; Internet is a global network of networks.
PAN个人，LAN建筑，MAN城市，WAN国家，Internet全球网络。
Five Layers
Application (message format), Transport (process-to-process, ports), Network (host-to-host, IP, routing), Link (hop-to-hop, MAC), Physical (bits on wire).
应用=消息格式，传输=进程通信+端口，网络=IP+路由，链路=逐跳MAC，物理=比特传输。
Encapsulation
Headers added going down sender stack, removed going up receiver stack.
发送向下加头，接收向上拆头。
FDM vs TDM
FDM divides frequency -> simultaneous. TDM divides time -> take turns.
FDM分频同时传，TDM分时轮流传。
Duplex
Simplex=one-way. Half-duplex=two-way not simultaneous. Full-duplex=simultaneous.
单工单向，半双工双向不同时，全双工同时双向。
Circuit vs Packet
Circuit reserves path -> predictable. Packet shares links -> efficient but variable delay.
电路预留可预测，分组共享高效但时延可变。
Four Delays
d_total = d_proc + d_queue + d_trans + d_prop. d_trans=L/R (length), d_prop=d/s (distance).
总时延=处理+排队+传输+传播。d_trans看包长，d_prop看距离。
ARP
Maps IP->MAC on same LAN. Request=broadcast(target unknown). Reply=unicast(requester known).
同LAN内IP转MAC。请求广播(目标未知)，应答单播(请求者已知)。
Switch
Learn source MAC. Lookup dest: known->unicast, unknown->flood, same port->filter.
学源查目：已知单播，未知泛洪，同端口过滤。
Subnetting
Borrow host bits for subnet. 2^x subnets, 2^h-2 usable hosts. Network(all-0) and broadcast(all-1) unusable.
主机借位给子网。2^x子网，2^h-2可用。全0网络和全1广播不可用。
Dijkstra
Link-state: know full topology. Pick closest unvisited, relax neighbors. D(v)=min(D(v),D(w)+c(w,v)).
链路状态：知全网拓扑。选最近未访问节点，松弛邻居。
Distance Vector
Bellman-Ford: share vectors only with neighbors. Distributed, iterative, async. D_x(y)=min{c(x,v)+D_v(y)}.
仅与邻居分享向量。分布式迭代异步更新。
IP vs MAC
IP=source->dest(end-to-end, unchanged). MAC=current hop sender->receiver(changes each hop).
IP=源到目的(不变)。MAC=当前跳收发(逐跳变)。
TCP Rules
1.SYN eats 1 seq. 2.FIN eats 1 seq. 3.Pure ACK eats 0. TCP numbers bytes, not segments.
1.SYN占1号 2.FIN占1号 3.纯ACK占0。TCP给字节编号。
TCP Handshake
1:Client SYN(Seq=x). 2:Server SYN+ACK(Seq=y,ACK=x+1). 3:Client ACK(Seq=x+1,ACK=y+1). Connected!
1.Client SYN 2.Server SYN+ACK 3.Client ACK。连接建立！
HTTP
HTTP/1.1 persistent by default(keep-alive). GET /path HTTP/1.1. 200=OK, 404=Not Found.
HTTP/1.1默认持久。GET路径 HTTP/1.1。200成功，404未找到。
